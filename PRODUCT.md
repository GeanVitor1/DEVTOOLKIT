# DevToolkit

## Register

product

## Platform

web

## Users

Desenvolvedores profissionais como público primário — usam o DevToolkit no dia a dia de trabalho para tarefas como depurar APIs, formatar JSON, decodificar JWTs, escrever SQL, testar regex e gerar UUIDs. Um público secundário de entusiastas e estudantes de tecnologia que buscam ferramentas gratuitas e confiáveis.

## Product Purpose

Seis ferramentas de desenvolvimento essenciais em um só lugar: HTTP Client, JSON Formatter, JWT Decoder, SQL Formatter, Regex Tester e UUID Generator. Cada ferramenta foi projetada para substituir múltiplos sites e aplicativos avulsos, oferecendo uma experiência profissional, 100% client-side e com privacidade total. Qualidade acima de quantidade — poucas ferramentas, qualidade máxima.

## Positioning

Seis ferramentas essenciais para desenvolvedores, gratuitas, 100% client-side e com qualidade de software comercial — tudo em um só lugar.

## Brand Personality

Profissional, preciso, confiável. Tom direto sem ser frio. Três palavras: **sharp, minimal, capable**. A interface deve passar a confiança de uma ferramenta que você usaria no trabalho, sem firulas — mas com o cuidado visual de um produto premium.

## Anti-references

- Não parecer um gerador de landing pages genérico com cards idênticos
- Nada de "glassmorphism", gradientes decorativos ou métricas genéricas
- Evitar o visual "SaaS cliché" com badges uppercase tracking em toda seção
- Ferramentas devem parecer profissionais, não "brinquedos online"

## Design Principles

1. **Ferramentas first.** A UI some na tarefa. Cada tool deve carregar rápida, responder instantaneamente e ocupar o espaço que precisa — sem decoração que atrapalhe o fluxo.
2. **Identidade por ferramenta.** Cada tool tem sua própria cor de destaque (HTTP ciano, JSON violeta, JWT rosa, SQL verde, regex laranja, UUID ciano claro). Essas cores são usadas com propósito: indicar qual tool está ativa, destacar elementos específicos, criar wayfinding visual.
3. **Consistência de vocabulário.** Mesmo estilo de botão, mesmo formato de input, mesma família tipográfica em todas as telas. A familiaridade é uma feature.
4. **Claro sobre clever.** Preferir labels diretos, ações nomeadas pelo que fazem, feedback imediato. Zero fricção.

## Accessibility & Inclusion

- Contraste ≥4.5:1 para texto body, ≥3:1 para texto grande
- prefers-reduced-motion respeitado
- Navegação por teclado funcional em todas as tools
- Foco visível em todos os elementos interativos
